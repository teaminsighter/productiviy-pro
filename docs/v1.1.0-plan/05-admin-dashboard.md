## Data Sources

| Metric | Source |
|--------|--------|
| User signups | Backend database |
| Active users | Backend (last activity timestamp) |
| Downloads | GitHub Releases API |
| Revenue/MRR | Stripe API |
| Landing analytics | Plausible/Umami (self-hosted) |
| Conversion funnel | Mix of above |

---

## Implementation Options

### Option 1: Separate Admin App (Recommended)
- New Next.js app in `apps/admin/`
- Shares backend with main app
- Deploy separately

### Option 2: Route in Landing Page
- Add `/admin` route to landing page
- Simpler deployment
- Same codebase

### Option 3: Route in Backend
- Server-rendered admin pages
- FastAPI + Jinja templates
- All-in-one backend

**Recommendation:** Option 1 - Separate admin app for cleaner separation.

---

## API Endpoints for Admin

```
GET  /api/admin/stats/overview        - Dashboard metrics
GET  /api/admin/stats/downloads       - Download statistics
GET  /api/admin/stats/revenue         - Revenue metrics (Stripe)
GET  /api/admin/users                 - List all users (paginated)
GET  /api/admin/users/:id             - User details
PUT  /api/admin/users/:id             - Update user (suspend, etc.)
GET  /api/admin/workspaces            - List all workspaces
GET  /api/admin/workspaces/:id        - Workspace details
GET  /api/admin/subscriptions         - Subscription list
```

## Download Tracking

### Option 1: GitHub API (Simple)
```python
# Fetch download counts from GitHub Releases
async def get_download_stats():
    response = await github_api.get(
        f"/repos/{RELEASES_REPO}/releases"
    )
    downloads = {}
    for release in response:
        for asset in release['assets']:
            downloads[asset['name']] = asset['download_count']
    return downloads
```

### Option 2: Custom Tracking (Advanced)
- Redirect downloads through your server
- Track IP, country, timestamp
- More detailed analytics
